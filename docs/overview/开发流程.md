# 开发流程

## 标准顺序

`需求输入 -> 产品收敛 -> 决策固化 -> 原型表达 -> 实现开发 -> 测试验收 -> 发布 -> 运营迭代`

## 人和 AI Agent 的工作顺序

1. 先读 `docs/overview/项目导航.md`
2. 再读 `docs/overview/Claude-Skill使用指南.md`
3. 需要业务背景时看 `docs/requirements/`
4. 需要执行开发任务时看 `docs/product/`
5. 存在冲突时先查 `docs/decisions/`
6. 需要看页面流转和界面表达时看 `design/`
7. 最后才进入 `apps/`

## 当前治理阶段约束

- 先统一结构和术语，再继续功能开发
- 未经过 `docs/product/` 明确认定的内容，不直接作为实现依据
- 实现有冲突时，以 `docs/product/` 和 `docs/decisions/` 为准
- 原型与 PRD 冲突时，以 `docs/product/` 为准

## Claude 协作建议

- 先用 `writing-plans` 拆方案，再做实现
- 开发小程序页面时优先使用 `uniapp-project`
- 有报错先走 `systematic-debugging`
- 提交前至少做一次 `requesting-code-review` 或 `verification-before-completion`
- 常用提示词模板统一放在 `prompts/claude-skill-调用模板.md`
