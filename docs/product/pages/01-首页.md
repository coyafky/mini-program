# 01-首页

## 页面定位

首页是蓝辉轻改小程序的品牌入口页，也是预约链路的第一触点。根据 PRD，这个小程序第一阶段的核心目标不是线上支付成交，而是帮助用户完成：

`了解项目 -> 建立品牌信任 -> 选择门店 -> 浏览商品/套餐 -> 发起预约`

因此，首页的定位不应是“纯电商首页”，而应定义为：

`以品牌展示、热门项目导购和门店引流为核心的预约型首页`

---

## PRD 对首页的约束

根据 [`../PRD-蓝辉轻改小程序.md`](../PRD-蓝辉轻改小程序.md)，首页需要重点满足以下要求：

- 展示 Banner 轮播图，且可由总部后台配置
- 展示热门车型入口
- 展示热门商品推荐
- 展示门店入口
- 提供搜索入口
- 支持首页内容排序、上下线和配置生效
- 用户在首页建立兴趣后，应被顺滑引导到门店和商品浏览链路

同时，PRD 的用户故事里还提到首页要承接“案例”内容，因此首页除了商品导购，也要承担一定的案例种草作用。

---

## 页面目标

- 建立品牌专业感和轻改装信任感
- 让用户快速看到当前主推项目、主推车型和主推活动
- 引导用户选择门店，进入门店商品浏览与预约链路
- 通过案例和专题内容降低用户决策门槛

---

## 目标用户

- 刚接触蓝辉轻改品牌、想先了解项目内容的车主
- 已有轻改装意向、希望快速浏览热门项目和套餐的用户
- 从短视频、私域或线下物料导流进入小程序的潜在客户

---

## 页面线框图

首页线框需要服务于 PRD 中的“预约到店型商城”目标，建议结构如下：

```text
┌──────────────────────────────────────┐
│ 顶部搜索栏                           │
│ [ 搜索商品 / 套餐 / 案例 ]           │
├──────────────────────────────────────┤
│ 主视觉 Banner 轮播区                │
│ [品牌活动 / 主推车型 / 到店利益点]   │
├──────────────────────────────────────┤
│ 热门车型入口                         │
│ [车型1] [车型2] [车型3] [更多]       │
├──────────────────────────────────────┤
│ 热门项目 / 热门商品推荐              │
│ [项目卡] [项目卡] [项目卡]           │
├──────────────────────────────────────┤
│ 热门套餐推荐                         │
│ [套餐卡] [套餐卡]                    │

└──────────────────────────────────────┘
           底部导航栏
```

## 线框说明

### 1. 顶部搜索栏

首页顶部提供统一搜索入口，支持搜索商品、套餐和案例。

作用：

- 给目标明确的用户提供快速检索入口
- 缩短“知道想要什么”用户的操作路径

交互说明：

- 点击跳转搜索页
- 搜索结果可按商品、套餐、案例分组展示

### 2. 主视觉 Banner

Banner 是首页首屏最重要的品牌与活动曝光区，必须支持后台配置。

可承载内容：

- 品牌主张
- 热门车型专题
- 门店活动
- 预约福利

交互说明：

- 支持轮播、点击跳转
- 可跳转到专题页、商品列表页、门店页或案例页

### 3. 热门车型入口

PRD 明确要求首页展示热门车型入口，因此这里应作为首页的一级导购模块。

作用：

- 帮助用户快速按车型进入浏览
- 缩短从首页到“适配商品列表”的路径

交互说明：

- 点击车型进入对应的商品/套餐专题列表页

### 4. 热门项目 / 热门商品推荐

这里承接 PRD 中“热门商品推荐”的要求，用于展示当前转化价值最高的改装项目。

展示内容：

- 主推商品
- 高点击项目
- 新品项目

交互说明：

- 点击进入商品详情页
- 若用户尚未选门店，则在详情或进一步操作前引导选择门店

### 5. 热门套餐推荐

套餐在 PRD 中是独立管理对象，因此首页建议单独给出套餐推荐模块，而不是完全混在商品流里。

作用：

- 快速展示标准化搭配方案
- 提升用户对“轻改装组合方案”的理解

交互说明：

- 点击进入套餐详情页
- 套餐详情页视为商品详情页的一种

### 6. 改装案例 / 专题内容

PRD 的用户故事中明确提到“案例”，因此首页不应省略种草内容。

作用：

- 增强品牌专业度
- 帮助用户通过真实效果理解项目价值
- 作为内容种草入口降低决策门槛

交互说明：

- 点击案例进入案例详情页
- 点击专题进入专题页或专题商品列表页

### 7. 门店入口

这是首页中必须重点保留的模块，因为 PRD 明确提出用户在浏览商品前需要先选择服务门店。

作用：

- 引导用户进入门店选择流程
- 让用户理解后续商品和预约都与门店绑定

交互说明：

- 点击进入门店列表页
- 支持查看附近门店或全部门店

---

## 信息层级建议

首页建议遵循以下信息层级：

1. 先建立品牌认知和活动吸引
2. 再给用户车型/项目导购入口
3. 再补充案例种草内容
4. 最后把用户推向门店选择和预约链路

这意味着首页不是单纯按“卖货模块堆叠”，而是按“认知 -> 兴趣 -> 导购 -> 预约”的漏斗来组织。

---

## 页面状态说明

### 默认状态

- Banner 正常轮播
- 展示后台配置的热门车型、热门商品、热门套餐和案例内容

### 内容为空状态

- 某模块暂无配置时自动隐藏或显示空占位
- 避免首页出现大片空白

### 加载状态

- 首屏使用骨架屏覆盖 Banner、车型入口和卡片模块

### 未选门店状态

- 首页本身允许浏览
- 当用户尝试进入强门店绑定的详情或预约动作时，引导先选门店

---

## 视觉方向

首页应延续汽车后市场的专业感和品牌感，但不宜做成沉重电商首页。

建议视觉关键词：

- 品牌专业
- 项目清晰
- 门店可信
- 预约顺畅

可以保留高质量车型图、项目效果图和专题 Banner，但整体结构要清晰，避免信息密度过高。

---

## 产出结论

阅读 PRD 后，首页的结构应更明确地服务于预约闭环，而不是购物闭环。

最终建议的首页核心结构为：

`搜索栏 -> Banner -> 热门车型 -> 热门项目/商品 -> 热门套餐 -> 案例/专题 -> 门店入口`

如果后续继续出低保真原型，这一版结构会比“传统商城首页”更符合 PRD。

## 视觉规范

### 一、视觉风格

| 属性 | 规范 |
|------|------|
| 主色调 | 暖金色 `#D8A04A`（品牌识别色） |
| 辅助色 | 深金 `#8B6914`（价格/强调）、浅暖白 `#FAF7F2`（卡片背景）、深灰 `#2C2C2C`（正文） |
| 圆角 | 模块圆角 16–22px，卡片 14–18px，按钮 20–24px |
| 阴影 | 轻投影 `0 2px 12px rgba(0,0,0,0.08)`，不做过重 |
| 字体 | 正文字号 14px，标题 16–18px，价格 18px/700 |
| 风格关键词 | 专业 · 清爽 · 汽车服务感 · 可信赖，不做廉价促销风 |

### 二、页面结构与模块规格

#### 2.1 搜索栏

- 高度 44px，圆角 22px（全圆角胶囊形态）
- 背景 `#F5F5F5`，hover/聚焦时边框 `1px solid #D8A04A`
- placeholder 文案："搜索商品 / 套餐 / 案例 / 门店"
- 左侧搜索图标（24px），右侧可省略清除按钮

#### 2.2 Banner 轮播区（Banner 1）

- 轮播比例 **16:9**（建议尺寸 **750×422px**）
- 圆角 18px，超出部分裁切
- 单张大小上限 **150KB**，格式 JPG/WebP
- 指示器：当前页为圆角矩形条（宽 16px 高 4px），非圆点；未选中为细灰点
- 自动轮播间隔 4s，用户拖动后暂停自动播放

#### 2.3 热门车型入口（快捷选块）

- 布局：4列 Grid，间距 12px
- 每个选块：图标 24px + 文字 12px + 右下角小箭头
- 选块背景浅金 `rgba(216,160,74,0.1)`，hover 时加深
- 选中态：背景 `#FDF2E2`，边框 `1px solid #D8A04A`
- 长文本截断：单行，文字超长显示省略号

#### 2.4 热门项目 / 商品推荐（商品卡片）

- 布局：2列 Grid，间距 12px，卡片圆角 14px
- 图片区域 **1:1 正方**（建议尺寸 **400×400px**，≤80KB）
- 商品名称：2行截断，超长显示省略号
- 价格：18px/700/深金色，单位小一号
- 标签：角标/热卖/新品，背景暖金，字色白
- 点击区域覆盖整张卡片
- 加载态：骨架屏 shimmer 动画替代 spinner
- 图片加载失败：显示灰色占位图标（汽车轮廓 SVG），不阻塞页面

#### 2.5 Banner 2（第2Banner / 活动横条）

- 高度约 100px，宽度撑满，比例约 **3.5:1**
- 背景暖黄渐变（从 `#FDF2E2` 到 `#F5E6C8`）
- 可承载"限时活动 / 预约福利"等短文案
- 圆角 12px，文字居左或居中

#### 2.6 改装案例 / 专题（横向滚动）

- 横向滚动容器，最小单元宽 180px，高 110px（**4:3** 比例）
- 封面图建议尺寸 **360×270px**，≤80KB，格式 JPG/WebP
- 案例标题：2行截断，14px，字色深灰
- 滚动时无惯性弹跳，滚动条隐藏
- 空态：整块隐藏，不显示占位

#### 2.7 门店入口

- 图标区 48×48px，背景 `rgba(216,160,74,0.14)`
- 文字：门店名称 + 距离，14px
- 主按钮："查看附近门店"（主色调），副按钮："全部门店"（文字按钮）
- 点击整块跳转门店列表页

#### 2.8 底部 Tabbar

- 固定在页面底部，height 56px，背景白色 + top border `1px solid #F0F0F0`
- 图标 24px + 文字 10px，当前页图标为主色
- 主色调强调当前 tab，灰阶未选中
- 背景可加毛玻璃效果 `backdrop-filter: blur(8px)`

### 三、状态定义

| 状态 | 表现 |
|------|------|
| **加载态** | 骨架屏 shimmer 动画覆盖各模块，不使用 spinner |
| **空态** | 对应模块整块隐藏，不显示占位符 |
| **错误态** | 图片加载失败 → 显示灰色汽车轮廓 SVG 占位，不阻塞页面其余内容 |

### 四、素材规格汇总

### 六、验收要点（前端 Checklist）

- [ ] 移动端 375px 宽度下文字不溢出、图片比例稳定
- [ ] 骨架屏 shimmer 动画替代 spinner
- [ ] 图片加载失败有灰色 SVG 占位图标兜底
- [ ] Banner 指示点当前页为圆角条（非圆点）
- [ ] 商品名称 2 行截断，显示省略号
- [ ] 车型选块切换有选中高亮态
- [ ] 门店入口整块可点击
- [ ] 横向滚动案例无惯性弹跳，滚动条隐藏
- [ ] 空态模块整块隐藏
- [ ] 后台发布后前台内容即时生效
- [ ] 首屏骨架屏在 1.5s 内消失（数据返回后）

### 七、关键开发约束

1. **不改变预约主链路**：所有跳转需引导用户先选门店，再进入商品/套餐详情
2. **图片比例强制统一**：商品图必须 1:1，案例封面必须 4:3，禁止自由比例
3. **骨架屏优先于 loading spinner**：首屏加载必须用骨架屏，不使用传统 loading 动画
4. **模块可独立上下线**：后台配置某模块内容为空时，前台对应模块整块不渲染
5. **移动端优先**：所有样式以 375px 宽为基准，右侧 padding 不少于 16px
