# 提交成功页 - 设计提示词

## 页面概述

用户预约提交成功后的确认页面。

## 页面结构

```
┌─────────────────────────────────────┐
│                                     │

│           ✓ (成功图标)               │
│                                     │
│         预约提交成功                  │
│                                     │
│      预约单号：YY202605130001        │
│                                     │
├─────────────────────────────────────┤
│                                     │
│  ┌─────────────────────────────┐   │
│  │ 门店信息                    │   │
│  │ 门店名称                    │   │

│  │ 📞 电话                     │   │

```

## 功能要求

| 模块 | 功能描述 | 交互要求 |
|------|----------|----------|
| 成功图标 | 视觉确认 | 动画可选 |
| 预约单号 | 显示唯一单号 | 可复制 |
| 门店信息 | 显示预约门店详情 | 地址+电话 |
| 预约信息 | 显示预约项目和时间 | 摘要展示 |
| 提示文字 | 告知用户等待联系 | 温馨提醒 |
| 查看预约 | 跳转预约详情 | 按钮 |
| 返回首页 | 跳转首页 | 按钮 |

## 预约单号规则

- 格式：`YY + 年月日 + 序号`
- 例如：`YY202605130001`
- 可点击复制

## 设计风格

- **风格**: 成功确认，简洁大气
- **色调**: 白色背景 + 绿色成功色 + 橙色按钮
- **成功图标**: 圆形绿色背景 + 白色对勾
- **信息卡片**: 灰色背景卡片，信息清晰

## 组件清单

| 组件 | 状态 | 说明 |
|------|------|------|
| 成功图标 | 静态/动画 | 绿色圆形+对勾 |
| 单号显示 | 默认/已复制 | 可复制 |
| 信息卡片 | 默认 | 门店+预约信息 |
| 提示文字 | 默认 | 居中显示 |
| 按钮组 | default/hover | 双按钮 |

## 技术要求

- 页面不可返回刷新
- 复制功能使用系统能力
- 按钮点击区域足够大

## 输出要求

生成可交互的 Figma/原型稿，包含：
- [ ] 成功状态视觉呈现
- [ ] 预约单号展示和复制
- [ ] 信息卡片布局
- [ ] 双按钮交互
- [ ] 整体页面氛围
