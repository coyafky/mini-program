# 04-门店商品页

## 页面定位

阅读 PRD 后，这个页面的定位需要更明确地回到业务主线。

蓝辉轻改小程序第一阶段的核心闭环是：

`用户线上浏览 -> 选择门店 -> 浏览当前门店可售商品/套餐 -> 查看详情 -> 发起预约`

因此，门店商品页虽然界面上看起来像“门店商城”，但在产品定义上不应该写成传统支付电商页，而应定义为：

`当前门店可预约商品/套餐的浏览与筛选页`

它的重点不是支付结算，而是：

- 确认当前服务门店
- 仅展示该门店可承接的项目/套餐
- 通过品牌、车型、分类帮助用户快速筛选
- 引导进入商品详情和预约流程

---

## PRD 对本页的约束

根据 [`../PRD-蓝辉轻改小程序.md`](../PRD-蓝辉轻改小程序.md)，本页至少需要满足：

- 只展示当前所选门店已上架商品
- 支持按品牌筛选
- 支持按车型筛选
- 支持按项目分类筛选
  分类包括：内饰、外观、功能、舒适、套餐
- 商品列表展示图片、名称、价格说明、简要介绍
- 点击商品进入商品详情页
- 未选门店时，不应允许进入需门店绑定的深层购买/预约链路

这意味着之前如果把页面写得过于“购物车结算化”，会偏离 PRD。

## 页面目标

- 让用户确认自己正在浏览哪一家门店
- 让用户看到该门店真实可预约的商品和套餐
- 让用户通过品牌、车型、项目分类快速定位内容
- 让用户从列表进入商品详情，并继续发起预约

---

## 目标用户

- 已在首页或门店列表中选定门店的用户
- 已经有明确车型或品牌倾向的车主
- 需要先看当前门店可承接哪些项目和套餐的用户

## 线框说明

### 1. 顶部搜索栏

搜索栏用于在当前门店内搜索商品、套餐或项目关键词。

作用：

- 降低用户在大量门店商品中的查找成本
- 保持与首页搜索体验一致

交互说明：

- 搜索结果默认限定在当前门店的可售范围内
- 支持搜索商品名、套餐名、项目关键词

### 2. 当前门店信息区

这是页面最关键的上下文模块，必须明确告诉用户“你现在看到的是哪家门店的服务内容”。

建议展示：

- 门店名称
- 距离
- 营业时间
- 履约方式切换

交互说明：

- 点击门店名称可进入门店详情或切换门店
- 切换门店后，整个商品列表和可预约内容都要刷新

### 3. 顶部筛选横栏

参考图中的“套餐权益包 + 标签切换”可以保留，但文档里要把它解释为“当前主题筛选”，而不是单纯营销导航。

作用：

- 在当前门店下进一步限定套餐主题或活动主题
- 承接运营配置的推荐内容

交互说明：

- 点击标签切换对应商品池/套餐池
- 展开按钮可查看更多主题

### 4. 左侧品牌 / 车型筛选栏

PRD 明确要求支持按品牌和车型筛选，因此左侧栏不只是“车型升级列表”，它应被定义为品牌/车型导购入口。

参考图中的分类文字可以保留，但文档应理解为：

- 某品牌线的升级集合
- 或某车型体系下的入口

交互说明：

- 点击后，右侧只展示该品牌/车型下、当前门店已配置的商品/套餐

### 5. 右侧商品 / 套餐列表

这一部分是页面主体，负责展示当前门店实际可预约内容。

根据 PRD，每个列表项至少需要有：

- 图片
- 名称
- 价格说明
- 简要介绍

如果参考图中只有标题和价格，也建议在文档规范中补上“简要介绍”字段要求，方便后续原型与开发一致。

交互说明：

- 点击列表项进入商品详情页
- 商品详情页展示当前门店信息，并提供“加入预约单”入口

### 6. 底部预约操作栏

这里是最需要按 PRD 修正的地方。

参考图里看起来像“购物车 + 去结算”，但 PRD 第一阶段并没有支付功能，核心动作是预约。因此文档不建议直接定义成支付结算栏，而建议改成：

- 已选项目汇总栏
- 去预约按钮

如果视觉上仍保留购物车图标，也建议在产品语义上把它理解为：

`预约单暂存区 / 已选项目区`

交互说明：

- 未选商品时，按钮置灰
- 已选商品后，可进入预约单提交页
- 预约单提交页收集手机号、车型、门店、意向项目、到店时间、备注

### 7. 底部 TabBar

当前页高亮“门店”，与首页、分类、购物车、我的形成主导航闭环。

其中“购物车”在 PRD 第一阶段里如果保留，也更适合理解为“已选项目/预约单入口”，而不是支付购物车。

## 筛选逻辑建议

门店商品页建议支持三层筛选语义：

1. 当前门店
2. 品牌 / 车型
3. 项目分类 / 套餐主题

其中：

- 门店是第一层强约束
- 品牌/车型帮助用户定位适配内容
- 分类和套餐标签帮助用户快速细化浏览范围

---

## 用户操作路径

典型路径如下：

1. 用户已选择门店并进入当前门店商品页
2. 查看门店名称、距离和营业时间
3. 按需切换自提/配送
4. 选择品牌/车型或套餐主题
5. 浏览当前门店可预约的商品/套餐
6. 点击列表项进入详情页
7. 在详情页发起预约或加入预约单

---

## 页面状态说明

### 默认状态

- 展示当前门店的默认推荐主题
- 展示第一个品牌/车型分类下的内容

### 未选门店状态

- 不建议直接进入本页
- 若通过异常入口进入，应先弹出门店选择引导

### 筛选切换状态

- 顶部主题筛选与左侧品牌/车型筛选均应实时更新右侧列表

### 空状态

- 当前门店在某品牌/车型或某主题下暂无可售项目时，展示空提示

### 异常状态

- 门店信息加载失败、商品列表失败时，应支持局部重试

---

## 产出结论

读完 PRD 后，这一页最重要的调整不是视觉结构，而是产品语义。

参考图可以继续沿用，但文档定义要从：

`门店商城页 / 去结算`

调整为：

`当前门店可预约商品浏览页 / 去预约`

最终建议的页面主结构为：

`搜索栏 -> 当前门店信息 -> 顶部主题筛选 -> 左侧品牌/车型筛选 -> 右侧商品列表 -> 底部预约操作栏 -> TabBar`

这会比单纯照着截图写“购物车结算页”更符合你的 PRD。
