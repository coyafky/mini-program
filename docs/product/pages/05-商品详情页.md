# 05-商品详情页

## 页面定位

商品详情页是蓝辉轻改小程序里承接“浏览兴趣 -> 形成意向 -> 发起预约”的核心转化页。

根据 [`../PRD-蓝辉轻改小程序.md`](../PRD-蓝辉轻改小程序.md)，这一页不是纯电商支付详情页，而是：

`当前门店下商品/套餐的详细介绍与预约入口页`

它的任务不是推动用户立刻线上付款，而是帮助用户：

- 充分了解项目或套餐内容
- 确认适配车型、规格和价格说明
- 明确当前服务门店
- 选择规格或版本
- 将意向项目加入预约单，进入预约提交流程

同时，PRD 明确提出：

- 商品详情页展示主图轮播
- 商品详情页展示项目介绍、价格说明、适配车型、规格说明
- 商品详情页显示当前门店信息
- 商品详情页提供“加入预约单”入口
- 点击后进入预约单提交页
- 套餐详情页视为商品详情页的一种

因此这份文档需要同时覆盖：

- 单品详情页
- 套餐详情页
- 规格选择弹层

3. `规格选择弹层`
用于选择版本、材质、组合或具体配置项，然后确认加入预约单。

## 页面目标

- 让用户对当前项目/套餐形成清晰认知
- 让用户明确项目适配性、内容范围和价格区间
- 让用户在当前门店语境下完成规格确认
- 让用户通过“加入预约单”继续进入预约流程

---

## 目标用户

- 已从门店商品页点击进入详情的用户
- 对某个项目、套餐或升级权益包感兴趣的用户
- 需要确认车型适配、版本差异和服务内容后再预约的用户

## 页面结构说明

### 1. 顶部导航区

参考图中的顶部更接近轻导航样式，通常包含：

- 返回
- 菜单
- 可能的分享入口

作用：

- 允许用户从详情返回列表
- 支持分享项目或套餐给他人

交互说明：

- 返回上一页
- 点击菜单展开页面级操作
- 分享走微信标准分享能力

### 2. 主图轮播 / 首屏大图

这是 PRD 明确要求的模块，也是详情页建立信任感的核心。

在不同类型的详情页中，表现形式可以不同：

- 单品详情页：更偏图文轮播与细节展示
- 套餐详情页：更偏主视觉海报与整套方案展示

设计建议：

- 单品页可使用多图轮播
- 套餐页首屏可以采用大海报主图 + 指示点
- 图片优先展示改装效果、项目卖点、使用场景

交互说明：

- 支持左右滑动
- 点击大图可进入预览模式

### 3. 价格与标题信息区

这是详情页最核心的信息密集区，必须在首屏内清晰展示。

建议包含：

- 当前价格或价格说明
- 商品 / 套餐全称
- 库存
- 销量

PRD 中要求“价格说明”，因此这里不一定都是单一价格，也可能是：

- 固定价格
- 起步价
- 价格区间
- 套餐版本价

交互说明：

- 标题区不承担复杂操作
- 价格和库存受当前规格选择影响时，需要联动更新

### 4. 当前门店信息区

PRD 明确要求商品详情页显示当前门店信息，因此这个区块不能省略。

建议展示：

- 当前服务门店名称
- 距离
- 营业时间
- 查看门店详情入口

作用：

- 强化“当前看到的是哪家门店可承接的服务”
- 为后续预约提交提供明确上下文

交互说明：

- 点击门店信息可查看门店详情或切换门店
- 若切换门店，详情内容需重新校验是否可售

### 5. 规格选择入口

这是这次文档里必须补强的重点。参考图 `商品细节选择-页.png` 已经很明确展示了规格弹层。

因此详情页不应该只写“规格说明”，还必须写清楚：

- 规格选择入口
- 弹层结构
- 选中状态

可选规格示例：

- 套餐版本
- 材质
- 颜色
- 组合方案
- 玻璃膜类型
- 前后排配置

交互说明：

- 点击规格区，弹出底部或居中弹层
- 必须选择完整规格后才能确认加入预约单
- 当前选中的规格摘要展示在入口行

### 6. 适配车型区

PRD 要求展示适配车型，因此详情页里必须有明确的车型适配说明。

展示建议：

- 标签式展示
- 首屏展示核心车型
- 更多车型可展开

交互说明：

- 点击查看更多适配车型
- 若当前用户已选择车型，可高亮适配结果

### 7. 产品介绍 / 图文详情

这一部分是单品详情页里最重要的说服区，也是参考图一里占比最大的部分。

应包含：

- 产品介绍
- 使用场景图
- 细节特写
- 卖点说明
- 材质/工艺说明
- 安装效果图

套餐详情页中则更偏：

- 套餐核心权益
- 套餐包含项目
- 升级效果说明

交互说明：

- 支持整页纵向滚动浏览
- 图文详情可以为富文本或结构化图片区块

### 8. 服务说明区

这一块虽然截图里不一定非常显性，但从业务上必须存在。

建议展示：

- 标准安装服务说明
- 到店预约说明
- 施工须知
- 售后或质保说明

作用：

- 缓解用户对服务交付的不确定感
- 增强到店预约的可信度

### 9. 底部操作栏

这是当前页面最需要按 PRD 修正语义的地方。

参考图里出现了：

- 首页
- 收藏
- 购物车
- 加入购物车
- 门店购买

但 PRD 第一阶段的产品主线不是支付结算，而是预约。因此文档建议这样统一解释：

- `购物车` 更接近“预约单暂存区”
- `加入购物车` 更接近“加入预约单”
- `门店购买` 更接近“门店预约 / 立即预约”

建议保留参考图的视觉布局，但在产品定义上改成：

- 首页
- 收藏
- 预约单
- 加入预约单
- 门店预约

交互说明：

- 点击“加入预约单”时，若未选规格，先弹规格层
- 点击“门店预约”时，若未选规格，也先完成规格选择
- 确认后进入预约单提交页

## 页面类型拆分

### A. 单品详情页

适用于：

- 头枕
- 玻璃膜
- 车顶行李箱
- 地板
- 单个内饰件/外观件

特点：

- 更强调图文细节
- 更强调具体规格和材质选择
- 图文详情内容较长

### B. 套餐详情页

适用于：

- 升级权益包
- 多项目组合套餐
- 车型专属升级套装

特点：

- 更强调整套方案价值
- 更强调套餐名称、版本、整体价格
- 图文详情更偏主题海报和套餐说明

结论：

套餐详情页不需要独立成完全不同的页面模板，但需要在同一详情页骨架下支持不同内容比重。

## 规格选择弹层说明

### 弹层目标

- 让用户在进入预约动作前先确认具体版本
- 避免预约单里出现模糊项目描述

### 弹层内容

- 商品缩略图
- 商品标题
- 当前价格
- 当前库存
- 规格组选项
- 确认按钮

### 交互规则

- 默认不选中完整规格时，确认按钮不可用或提示补选
- 选择不同规格后，价格和库存可联动变化
- 点击确认后，回填详情页已选规格摘要

---

## 页面状态说明

### 默认状态

- 正常展示首图、价格、标题、当前门店、规格入口和详情内容

### 未选门店状态

PRD 里写明“未选门店时无法进入商品详情页”，所以理论上这个页面应建立在已选门店前提下。

但为了兜底，若异常进入：

- 页面应提示先选择门店
- 禁止加入预约单与预约操作

### 未选规格状态

- 可以浏览详情
- 不可直接完成加入预约单
- 点击 CTA 时先唤起规格选择弹层

### 规格已选状态

- 展示已选规格摘要
- CTA 可正常执行

### 内容加载失败状态

- 首图、详情、门店信息可分区重试
- 避免整页空白

---

## 数据字段建议

基于 PRD，这页至少需要承接以下数据：

- 商品/套餐名称
- 主图列表
- 价格说明
- 商品介绍
- 规格说明
- 适配车型
- 当前门店信息
- 库存
- 销量
- 规格组与规格值

若是套餐类型，还应额外支持：

- 套餐包含项目列表
- 套餐适用车型

---

## 用户操作路径

### 标准路径

1. 用户从门店商品页进入详情页
2. 浏览主图、标题、价格和门店信息
3. 查看产品介绍、适配车型与规格说明
4. 点击规格入口选择具体版本
5. 点击“加入预约单”或“门店预约”
6. 进入预约单提交页

### 套餐路径

1. 用户进入套餐详情页
2. 浏览套餐海报、价格和套餐介绍
3. 选择套餐版本
4. 发起预约

---

## 产出结论

这页不应再被写成一个泛泛的“商品详情提示词”，而应该被定义为：

`当前门店下商品/套餐的详细介绍与预约转化页`

最终页面结构建议固定为：

`顶部导航 -> 主图轮播/海报 -> 价格标题区 -> 当前门店信息 -> 规格选择入口 -> 适配车型 -> 产品介绍/图文详情 -> 服务说明 -> 底部预约操作栏`

同时，必须把 `规格选择弹层` 一并纳入页面设计说明，因为它已经是参考图中的真实交互，也是后续预约链路成立的关键步骤。
