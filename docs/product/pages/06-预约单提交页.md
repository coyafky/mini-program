# 预约单提交页 - 设计提示词

## 页面概述

用户确认预约信息并提交的表单页面。

│  └─────────────────────────────┘   │
│                                     │
│  ┌─────────────────────────────┐   │

## 功能要求

| 模块 | 功能描述 | 交互要求 |
|------|----------|----------|
| 返回 | 返回商品详情 | 顶部导航 |
| 预约商品 | 显示当前预约商品 | 不可修改 |
| 当前门店 | 显示预约门店 | 可更换门店 |
| 手机号 | 必填，收集联系方式 | 格式校验 |
| 车型 | 必填，用户车辆信息 | 输入/选择 |
| 到店时间 | 必填，预约日期 | 日期选择 |
| 备注 | 可选，补充说明 | 多行文本 |
| 提交预约 | 核心CTA按钮 | 提交表单 |

## 表单校验规则

| 字段 | 规则 | 错误提示 |
|------|------|----------|
| 手机号 | 必填，11位手机号 | "请输入正确的手机号" |
| 车型 | 必填，非空 | "请输入您的车型" |
| 到店时间 | 必填，不能早于今天 | "请选择到店时间" |

## 日期选择

- 仅支持选择今天及以后的日期
- 显示格式：YYYY-MM-DD
- 可选择的时间范围：未来30天

## 设计风格

- **风格**: 清晰表单，简单易填
- **色调**: 白色背景 + 灰色输入框边框 + 橙色提交按钮
- **输入框**: 浅灰边框，圆角输入框
- **必填标识**: 红色星号
- **提交按钮**: 满宽橙色按钮

## 组件清单

| 组件 | 状态 | 说明 |
|------|------|------|
| 商品卡片 | 默认 | 不可编辑 |
| 门店选择 | default/选择中 | 可切换 |
| 输入框 | default/focus/error | 表单输入 |
| 日期选择器 | default/open | 底部弹出 |
| 提交按钮 | default/loading/disabled | 提交表单 |
| 错误提示 | 显示/隐藏 | 输入框下方 |

## 技术要求

- 表单字段自动填充（如已登录用户信息）
- 输入框获取焦点时放大点击区域
- 提交时显示 loading 状态
- 提交成功后跳转成功页

## 输出要求

生成可交互的 Figma/原型稿，包含：
- [ ] 表单完整布局
- [ ] 各输入框状态（默认/聚焦/错误）
- [ ] 日期选择器交互
- [ ] 提交按钮状态（可用/加载中/禁用）
- [ ] 错误提示展示
