# 蓝辉轻改小程序 - 第一阶段开发任务清单

## 1. 项目目标

第一阶段目标不是支付商城，而是打通：

`浏览商品/套餐 -> 选择门店 -> 提交预约 -> 总部处理 -> 门店线下承接`

当前已确认的产品约束：

- 不做在线支付
- 用户存在两条并存路径：
  - 路径 A：先选商品，再选门店，再提交预约
  - 路径 B：先选门店，再看该门店商品，再提交预约
- `预约单` 的语义接近购物车，用于承载多个商品/套餐的预约意向
- `我的预约` 放在 `我的` 中，用于查看已提交预约记录
- 后台是总部运营后台，不是门店后台

## 2. 开发前对齐事项

这些内容建议在正式开发前先统一：

1. 页面命名统一
- 当前原型中已存在 `design/prototypes/mini-program/03-product-list.html` 和 `design/prototypes/mini-program/08-booking-list.html`
- 但文档目录里暂未单独拆出”商品列表页”和”我的预约页”
- 建议补两个文档：
  - `docs/product/pages/03-商品列表页.md`
  - `docs/product/pages/10-我的预约页.md` 或调整现有编号体系

2. 预约页语义统一
- 当前前台应拆成：
  - `预约单页`：当前意向商品集合
  - `我的预约页`：已提交预约记录
- 开发时不要再把这两者混成一个页面

3. 后台职责统一
- 门店不做独立后台
- 后台仅面向总部运营
- 门店商品可见范围由总部后台配置

## 9. 第一阶段验收标准

### 9.1 前台验收

- 用户可从首页、分类页进入商品列表和商品详情
- 用户可先看商品，再去选门店
- 用户也可先选门店，再看门店商品
- 用户可将多个商品/套餐加入预约单
- 用户可统一提交预约
- 用户可在“我的”中查看我的预约记录

### 9.2 后台验收

- 总部可管理门店
- 总部可管理商品和套餐
- 总部可配置首页内容
- 总部可查看并处理预约单
- 总部可查看基础统计数据

### 9.3 数据一致性验收

- 门店配置影响前台可见商品范围
- 商品/套餐上下架影响前台展示
- 预约单提交后后台可立即查看
- 后台状态更新后前台我的预约能看到最新状态
